# Release Evidence Card CLI Spec

## Source Candidate

- Candidate: release-evidence-card-cli-01
- Decision: build
- Result count: 259

## Wedge

A local CLI that runs from npx, detects an npm-based CLI project, performs pack/install smoke from the generated tarball in a temporary sandbox, checks release governance files, and emits a shareable release-evidence.md plus release-evidence.html in under five minutes.

## Commands

- `release-evidence-card check [--dir <target>]` — Check release evidence for target directory
- `release-evidence-card run --entry "<command>"` — Run pack/install smoke test
- `release-evidence-card demo` — Run demo against bundled fixtures

## Checks Performed

### Package Info
- Reads package.json
- Detects bin entry (validates target file exists)
- Detects Node engine constraint
- Reports private/npm publish status

### Governance Files
- Checks for: LICENSE, SECURITY.md, CONTRIBUTING.md, CHANGELOG.md
- Configurable via `governanceFiles` in `.release-evidence-card.json`
- Also checks `repository` field in package.json (reported as PASS/FAIL in governance section)

### CI Detection
- Checks for `.github/workflows/ci.yml` or `.github/workflows/ci.yaml`
- Configurable via `ciPaths` in `.release-evidence-card.json`

### README First-Run Score (5 criteria)
1. Target user mention
2. Install command
3. First-run command
4. Expected output
5. License mention

### Configuration
- Optional `.release-evidence-card.json` in target directory
- Fields: `ignoreDirs`, `governanceFiles`, `ciPaths`, `readmePaths`
- Missing/invalid config falls back to defaults
- Default ignoreDirs: node_modules, .git, .github

### Smoke Test
- Runs `npm pack` in target directory
- Installs tarball in temporary sandbox
- Executes entry command
- Captures and redacts output

## Fixtures (for demo)

- **passing**: Full governance files, valid bin entry, scored README, CI workflow
- **failing**: Missing governance files, bin entry points to missing file, 0/5 README
- **missing-bin**: No bin entry in package.json
- **passing-readme**: README scoring 5/5
- **weak-readme**: README scoring 1/5

## Output

- `release-evidence.md` — Markdown report with package info, governance file status, CI detection, README score, configuration summary, and PASS/PENDING verdict
- `release-evidence-smoke.md` — Smoke test report (generated by `run` command)

## Safety

- Local-only: no network calls during check or demo operations
- No GitHub writes, repo creation, issues, PRs, releases, or packages
- All operations run on local filesystem

## Blockers

- None recorded.

## Acceptance

- Reads local files and produces evidence reports.
- Checks governance files, CI workflow, package.json, and README completeness.
- Runs pack/install smoke in isolated temporary sandbox.
- Supports config file for customizing checks.
- Includes bundled fixtures for demo and failure-path testing.
- Makes no external network or GitHub write calls.
- All checks pass against the project's own source.
